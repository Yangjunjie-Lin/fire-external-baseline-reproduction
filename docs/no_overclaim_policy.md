# No-overclaim Policy

## Accepted phrases

- E-KELL-style paper-faithful pipeline-level reimplementation
- external baseline reproduction scaffold
- independent baseline package
- system-level comparison baseline
- not official E-KELL reproduction
- proxy automatic metrics
- manual rubric required for paper-level claims
- structurally ready baseline package scaffold

## Forbidden phrases

Do not use these unless conditions are explicitly satisfied:

- official E-KELL reproduction
- reproduces E-KELL results
- certified emergency response
- proves safety
- top-tier ready results
- complete GraphRAG reproduction, unless actual package/indexing/query are implemented
- complete LightRAG reproduction, unless actual package/indexing/query are implemented
- final paper results, unless final experiments and analysis have been completed

## Required caveats

When discussing B2, say it is E-KELL-style and pipeline-level. When discussing automatic metrics, say they are proxies. When discussing final paper claims, say expert/manual evaluation and statistical analysis are required.
